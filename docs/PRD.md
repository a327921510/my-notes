# 产品需求文档（PRD）

**项目名称**：My Notes（暂定）  
**版本**：V1.0（MVP）  
**文档状态**：评审稿  
**日期**：2026-04-13  

---

## 1. 产品背景

用户在网页阅读、工作记录过程中有大量“即时摘录”需求，但现有工具要么偏重、要么流程中断明显。  
本产品希望先做一个**轻量、低打扰、本地优先**的笔记工具，支持文字+图片记录，并通过手动上传实现云端备份与跨端恢复。  
V1 的重点记录对象为**网页相关短文本数据**，包括账号、密码、ID、待办事项等，并以网页地址进行数据隔离与组织。  
该能力作为独立的**特殊功能区（站点信息区）**存在，与普通笔记区分开，避免混用。

---

## 2. 产品目标

### 2.1 核心目标

- 提供轻量笔记体验：快速记录文字与图片。
- 实现本地优先：Web 端离线可用，数据持久化到 `IndexedDB`。
- 实现可控同步：**推送（手动上传）**与**拉取（从云端合并到本地）**均由用户主动触发，不做自动实时同步。
- 支持**全量离线迁移**：将本地笔记正文与图片打包导出，在其它浏览器或设备上导入恢复。
- 重点支持网页短文本管理：账号、密码、ID、待办等高频信息的快速记录与检索。
- 按网页地址隔离数据：以 `domain/url` 作为默认分区维度，避免站点间信息混淆。

### 2.2 MVP 成功标准

- 用户可完成：创建笔记 -> 本地保存 -> 手动上传 -> 状态反馈。
- 支持文件夹 + 笔记结构管理。

## 3. 产品范围

### 3.1 本期范围（In Scope）

- Web 端（主产品）
- 浏览器扩展（网页短文本数据管理；**账号登录**后与 Web **共用同一云端账号与数据**；**手动推送 / 手动拉取**与 Web 语义一致）
- 后端（认证 + 上传接口 + 存储）

### 3.2 非本期范围（Out of Scope）

- PC 客户端
- 自动实时同步
- 浏览器扩展“一键保存整页网页”
- 富文本复杂排版（先不做，优先轻量编辑）

## 4. 目标用户与使用场景

### 4.1 目标用户

- 高频网页阅读与信息整理用户（开发、产品、学生、研究者）
- 需要“先记再整理”的轻量知识用户

### 4.2 核心场景

1. 用户在网页中快速收集文本片段，稍后在 Web 端整理。
2. 用户在 Web 端创建图文笔记，离线可用。
3. 用户仅在需要跨设备/备份时主动点击上传。
4. 用户在“站点信息区”中为不同网站分别维护账号、密码、ID、待办等短文本信息，互不干扰。

## 5. 关键概念定义

- **本地笔记**：仅存于浏览器 `IndexedDB` 的笔记数据。  
- **网页短文本条目（Snippet）**：与网页地址关联的短文本记录，典型内容为账号、密码、ID、待办事项。  
- **地址隔离（Site Isolation）**：按 `sourceDomain`（必要时精确到 `sourceUrl`）对数据进行逻辑分组与访问筛选。  
- **特殊功能区（站点信息区）**：独立于笔记模块的功能域，专门管理按站点隔离的短文本数据。  
- **推送（手动上传 / 同步上行）**：用户手动触发，将本地变更上传到云端。  
- **拉取（手动拉取 / 同步下行）**：用户手动触发，从云端获取数据并与本地合并（需登录；冲突策略见下文）。  
- **离线导出包**：将本地 `folders / notes / images(blobs) / snippets / site_spaces` 等结构化导出为单一文件或目录（含图片二进制或 Base64，格式在实现期确定），用于跨设备、跨浏览器迁移，不依赖云端。  
- **扩展短文本面板（Extension Snippets）**：浏览器扩展内的网页短文本管理界面，仅处理站点信息条目。  
- **同步状态（syncStatus）**：
  - `local_only`：仅本地
  - `synced`：已上传且本地与云端一致
  - `dirty`：曾上传但本地已修改
  - `failed`：最近一次上传失败

## 6. 功能需求

## 6.1 Web 端功能

### 6.1.1 账号与会话

- 用户可登录/退出。
- **登录入口**：Web 端 Header 右侧，未登录时显示「登录」按钮；点击弹出登录弹窗（`LoginModal`）。
- **登录弹窗**：Ant Design `Modal + Form`，包含邮箱（必填、格式校验）与密码（必填）两个字段；提交后调用 `POST /api/auth/login`，成功后写入 Zustand `useAuthStore`（persist 到 localStorage），弹窗关闭；Header 切换为用户邮箱 + 退出按钮。
- **退出**：点击 Header「退出」按钮，清除 token 与用户信息。
- 未登录用户也可本地记录；上传操作需登录后执行。

### 6.1.2 文件夹管理

- 新建、重命名、删除文件夹。
- 提供文件夹列表查询（可按名称排序 / 搜索）。
- 支持多级文件夹（树形结构）。
- 删除文件夹时可选择：
  - 同时删除子内容（软删除）
  - 或移动子内容到默认目录（可选后置）

### 6.1.3 笔记管理

- 新建、编辑、删除、移动笔记。
- 笔记字段：
  - 标题（可空时自动生成首行摘要）
  - 正文文本
  - 图片列表
  - 来源信息（可选）
  - 更新时间
  - 同步状态

### 6.1.3A 特殊功能区：站点信息区（重点）

- 支持创建短文本条目，字段建议包含：
  - 类型：`account` / `password` / `id` / `todo` / `custom`
  - 内容：纯文本（短内容）
  - `sourceDomain`（必填）
  - `sourceUrl`（可选）
- 条目默认进入对应 `sourceDomain` 分组。
- 支持对同一域名下条目进行新增、编辑、删除、搜索。
- 站点信息区与笔记区在信息架构和界面入口上独立呈现。
- 短文本条目默认不落入笔记树；仅在用户主动“转为笔记”时才进入笔记模块（后置可选）。

### 6.1.3B SitesPage 页面需求模板（可直接落地）

> 用途：本节作为 `apps/web/src/pages/SitesPage` 的实现与验收基线，覆盖信息架构、同步流程、冲突处理、异常与边界。  
> 页面定位：站点信息区的主工作台，负责“按站点组织短文本条目 + 本地与云端双向同步”。

#### A. 页面目标与角色

- 用户可在一个页面内完成：
  - 站点列表浏览与搜索；
  - 站点下短文本条目（账号/密码/ID/待办/自定义）CRUD；
  - 手动上行（本地 -> 云端）与手动下行（云端 -> 本地）；
  - 冲突识别、处理与结果确认。
- 页面角色边界：
  - **SitesPage** 只处理 `site_spaces + snippets` 域；
  - 不直接承担笔记树 `folders/notes` 的编辑职责。

#### B. 信息架构与分层（对应页面分层规范）

- 页面入口（`SitesPage/index.tsx`）：
  - 管理跨区域共享状态：`selectedSiteId`、筛选条件、同步任务状态、冲突队列；
  - 编排“列表区 <-> 详情区 <-> 同步弹窗”的事件流。
- 区域组件：
  - `SiteListPanel`：站点列表、搜索、创建站点、选择站点；
  - `SiteDetailPanel`：当前站点条目列表、条目编辑、删除、批量操作；
  - `SyncPanel`（可抽屉/弹窗）：展示上行/下行计划、冲突统计、执行结果。
- 业务 Hook：
  - `useSitesPageData`：本地读取、筛选、分页（若有）；
  - `useSitesSyncActions`：上行、下行、冲突处理动作；
  - `useSitesMutations`：站点与条目 CRUD。
- 纯展示组件：
  - 状态徽标、空态、错误态、字段展示行、冲突对比卡片等。

#### C. 核心实体与字段约束（SitesPage 范围）

- `site_spaces`（站点空间）：
  - `id`（本地主键，必填）
  - `sourceDomain`（必填，唯一约束：同用户同域名唯一）
  - `displayName`（可选，默认等于 `sourceDomain`）
  - `createdAt`、`updatedAt`
  - `cloudId`（可空；已同步后写入）
  - `syncStatus`（`local_only/synced/dirty/failed`）
- `snippets`（站点短文本）：
  - `id`（本地主键，必填）
  - `siteSpaceId`（必填）
  - `type`（`account/password/id/todo/custom`）
  - `content`（必填，建议 1~2000 字）
  - `sourceDomain`（必填，与所属站点一致）
  - `sourceUrl`（可选）
  - `createdAt`、`updatedAt`
  - `cloudId`（可空）
  - `syncStatus`（同上）
- 校验规则：
  - 不允许空 `sourceDomain` 入库；
  - `content` 去首尾空格后为空则不允许保存；
  - 同一 `siteSpaceId` 下支持同类型多条记录（不做唯一限制）。

#### D. 页面主流程（用户视角）

1. 进入 `SitesPage`，读取本地站点与条目；
2. 默认选中最近更新站点（或首个站点）；无数据则展示引导空态；
3. 用户可新增站点、编辑条目、删除条目；
4. 本地编辑后条目状态变为 `local_only` 或 `dirty`；
5. 用户手动执行：
  - `同步到云端`：上传 `local_only/dirty/failed`；
  - `从云端同步到本地`：拉取云端变更并合并；
6. 若出现冲突，进入冲突处理弹窗；处理完成后更新本地与状态标记。

#### E. 同步策略（必须明确）

##### E1. 同步到云端（上行 Push）

- 触发方式：用户手动点击，不做自动后台上行。
- 前置条件：已登录；未登录则引导登录后继续。
- 上行对象：`site_spaces` 与 `snippets` 中状态为 `local_only/dirty/failed` 的记录。
- 执行顺序（建议）：
  1. 先上行 `site_spaces`（确保远端存在站点容器）；
  2. 再上行 `snippets`；
  3. 批量成功后回写 `cloudId`、`syncedAt`（若有）并置 `syncStatus=synced`。
- 失败处理：
  - 失败记录置 `failed`；
  - 保留本地编辑内容；
  - 支持针对失败记录重试与“仅重试失败项”。

##### E2. 从云端同步到本地（下行 Pull）

- 触发方式：用户手动点击；登录成功后可自动触发一次“首次拉取”（可配置开关）。
- 拉取范围：当前用户云端 `site_spaces + snippets`（支持分页或增量游标）。
- 本地合并原则：
  - 云端有、本地无：新增到本地，标 `synced`；
  - 本地有、云端无：保持本地，不自动删除；
  - 双方都有且一致：保持 `synced`；
  - 双方都有且不一致：进入冲突判定（见 E3）。
- 下行失败：
  - 不清空本地；
  - 标记本次任务失败并显示失败原因；
  - 支持重试。

##### E3. 冲突定义与处理（MVP 推荐策略）

- 冲突判定（任一满足即冲突）：
  - 同一业务实体（同 `cloudId` 或同 `clientId` 映射）在云端与本地内容不同；
  - 且本地 `updatedAt` 与云端 `updatedAt` 均晚于“上次成功同步时间点”。
- MVP 默认策略：**Last-Write-Wins（LWW）+ 可审计保留**
  - 默认采用更新时间较新的版本为主版本；
  - 被覆盖版本写入本地冲突备份表（或冲突日志）用于追溯；
  - 页面给出“已自动解决 x 条冲突，可在冲突记录查看”提示。
- 用户可选增强（推荐开关，非阻塞）：
  - 对 `password/account` 类型冲突弹窗要求人工选择（本地版/云端版/合并后保存）；
  - 对 `todo/custom` 默认走 LWW 自动解。
- 冲突处理后状态：
  - 主版本 `syncStatus=synced`；
  - 备份版本标记 `conflicted_archived`（仅本地可见，不参与常规列表）。

#### F. 验收标准（DoD）

- 功能验收
  - 可创建、编辑、删除站点与站点条目，刷新后数据仍在；
  - 可按域名搜索站点、按类型筛选条目；
  - 可执行“同步到云端”并正确更新状态（`synced/failed`）；
  - 可执行“从云端同步到本地”，新增/更新数据正确落地；
  - 冲突场景下按策略产出可解释结果（自动或人工）。
- 状态验收
  - 任意一次本地编辑都会把记录变更为 `dirty`（或新建为 `local_only`）；
  - 上传成功后状态统一回到 `synced`；
  - 网络或鉴权失败不导致本地数据丢失。
- 交互验收
  - 页面具备加载态、空态、错误态；
  - 同步任务有进度与结果反馈（成功数、失败数、冲突数）；
  - 删除敏感记录（如密码）需二次确认。
- 数据一致性验收
  - 同一账号在两端并发修改后，冲突策略结果与 PRD 一致；
  - 重复点击同步不会产生重复云端记录（幂等）。

#### G. 边界条件与异常清单

- 登录边界：
  - 未登录允许本地 CRUD；
  - 未登录触发同步时仅提示登录，不执行网络写操作。
- 网络边界：
  - 同步中断（断网/超时）时，已成功项保留成功状态，未完成项保持原状态或标 `failed`；
  - 重试不应重复创建远端实体（依赖 `clientId` 幂等）。
- 数据边界：
  - 同域名站点重复创建时提示“已存在”，允许跳转到已有站点；
  - 超长文本、非法 URL、空内容需前置校验并阻止提交。
- 并发边界：
  - 同一浏览器多标签同时编辑同一条目时，以最后写入本地者为准，并在下次同步进入冲突判定；
  - 同步任务执行中再次点击同步，应做串行化（排队）或按钮禁用。
- 删除边界：
  - 本地删除后未上行前可撤销（可选）；
  - 已同步记录删除时需支持策略：
    - 仅本地删除（云端保留）；
    - 本地+云端同时删除（需明确确认）。

#### H. 测试用例最小集合（建议纳入提测单）

- 用例 1：新建站点与 3 条短文本 -> 上行成功 -> 状态全 `synced`。
- 用例 2：下行拉取云端新增站点 -> 本地正确新增。
- 用例 3：本地与云端同时改同一条 `password` -> 触发冲突并按策略处理。
- 用例 4：上行过程中断网 -> 失败项可重试且不重复创建。
- 用例 5：未登录点击同步 -> 不改本地数据，仅提示登录。
- 用例 6：执行“仅本地删除”与“本地+云端删除”分别验证结果。

### 6.1.3C CloudDrivePage页面需求（可直接落地）

> 页面定位：云盘的主工作台，负责“按文件目录组织附件 + 本地与云端双向同步”。

#### A. 页面目标与角色

- 用户可在一个页面内完成：
  - 云盘目录和文件的浏览与搜索；
  - 云盘目录和目录下文件CRUD；
  - 手动上行（本地 -> 云端）与手动下行（云端 -> 本地）；
  - 冲突识别、处理与结果确认。
- 页面角色边界：
  - **CloudDrivePage** 只处理 `drive_folders + drive_files` 域；
  - 不直接承担站点信息区 `site_spaces/snippets` 与笔记正文编辑职责；
  - 与编辑器页的边界为“附件管理”：本页负责文件对象与同步，不负责富文本内容排版与渲染。

#### B. 信息架构与分层（对应页面分层规范）

- 页面入口（`CloudDrivePage/index.tsx`）：
  - 管理跨区域共享状态：`selectedFolderId`、筛选条件、同步任务状态、冲突队列；
  - 编排“列表区 <-> 详情区 <-> 同步弹窗”的事件流。
- 区域组件：
  - `CloudDriveListPanel`：文件夹列表、搜索、创建文件夹、选择文件夹；
  - `CloudDriveDetailPanel`：当前文件夹列表、文件删除、文件新增、批量操作；
  - `SyncPanel`（可抽屉/弹窗）：展示上行/下行计划、冲突统计、执行结果。
- 业务 Hook：
  - `useCloudDrivePageData`：本地读取、筛选、分页（若有）；
  - `useCloudDriveSyncActions`：上行、下行、冲突处理动作；
  - `useCloudDriveMutations`：文件夹与文件 CRUD。
- 纯展示组件：
  - `CloudDriveDetailTopBar`: 展示文件夹路径、拉取云端、同步到云端、文件上传
  - 状态徽标、空态、错误态、字段展示行、冲突对比卡片等。

#### C. 核心实体与字段约束（CloudDrivePage 范围）

- `drive_folders`（云盘目录）：
  - `id`（本地主键，必填）
  - `name`（必填，建议 1~120 字）
  - `parentId`（可空；MVP 可仅支持单层目录，后续扩展树形）
  - `path`（可选缓存字段，如 `/工作/项目A`）
  - `createdAt`、`updatedAt`
  - `cloudId`（可空；已同步后写入）
  - `syncStatus`（`local_only/synced/dirty/failed`）
- `drive_files`（云盘文件）：
  - `id`（本地主键，必填）
  - `folderId`（必填，关联 `drive_folders.id`）
  - `name`（必填，建议 1~255 字）
  - `mimeType`（可选）
  - `sizeBytes`（必填，>=0）
  - `checksum`（可选，建议用于幂等与冲突判定）
  - `localPath`（可选，本地文件引用）
  - `cloudUrl`（可选，云端访问地址）
  - `createdAt`、`updatedAt`
  - `cloudId`（可空）
  - `syncStatus`（同上）
- 校验规则：
  - 同一 `parentId` 下目录名唯一（大小写不敏感，去首尾空格后比较）；
  - 同一 `folderId` 下文件名唯一（重复上传时给出“重命名/覆盖/取消”）；
  - 不允许空文件名、非法字符文件名（按平台约束过滤）；
  - 文件大小超限时前置拦截并给出可执行提示（按后端限制配置）。

#### D. 页面主流程（用户视角）

1. 进入 `CloudDrivePage`，读取本地文件夹与文件；
2. 默认选中首个文件夹；无数据则展示引导空态；
3. 用户可新增文件夹、删除文件夹、新增文件、删除文件；
4. 本地编辑后文件状态变为 `local_only` 或 `dirty`；
5. 用户手动执行：
  - `同步到云端`：上传 `local_only/dirty/failed`；
  - `从云端同步到本地`：拉取云端变更并合并；

#### E. 同步策略（规则与 6.1.3B E 一致）

##### E1. 同步到云端（上行 Push）

- 触发方式：用户手动点击，不做自动后台上行。
- 前置条件：已登录；未登录则引导登录后继续。
- 上行对象：`drive_folders` 与 `drive_files` 中状态为 `local_only/dirty/failed` 的记录。
- 执行顺序（建议）：
  1. 先上行 `drive_folders`（确保远端目录存在）；
  2. 再上行 `drive_files` 元数据；
  3. 最后上传文件二进制内容并回写 `cloudUrl`；
  4. 批量成功后回写 `cloudId`、`syncedAt`（若有）并置 `syncStatus=synced`。
- 失败处理：
  - 失败记录置 `failed`；
  - 保留本地文件与元数据，不做静默删除；
  - 支持针对失败记录重试与“仅重试失败项”。

##### E2. 从云端同步到本地（下行 Pull）

- 触发方式：用户手动点击；登录成功后可自动触发一次“首次拉取”（可配置开关）。
- 拉取范围：当前用户云端 `drive_folders + drive_files`（支持分页或增量游标）。
- 本地合并原则：
  - 云端有、本地无：新增到本地，标 `synced`；
  - 本地有、云端无：保持本地，不自动删除；
  - 双方都有且一致：保持 `synced`；
  - 双方都有且不一致：进入冲突判定（见 E3）。
- 下行失败：
  - 不清空本地；
  - 标记本次任务失败并显示失败原因；
  - 支持重试。

##### E3. 冲突定义与处理（MVP 推荐策略）

- 冲突判定（任一满足即冲突）：
  - 同一业务实体（同 `cloudId` 或同 `clientId` 映射）在云端与本地元数据不同；
  - 或文件内容 `checksum` 不一致，且本地与云端均有新修改；
  - 且本地 `updatedAt` 与云端 `updatedAt` 均晚于“上次成功同步时间点”。
- MVP 默认策略：**Last-Write-Wins（LWW）+ 可审计保留**
  - 默认采用更新时间较新的版本为主版本；
  - 被覆盖版本写入本地冲突备份表（或冲突日志）用于追溯；
  - 页面给出“已自动解决 x 条冲突，可在冲突记录查看”提示。
- 用户可选增强（推荐开关，非阻塞）：
  - 对同名且内容差异的文件，弹窗要求人工选择（保留本地/保留云端/另存副本）；
  - 对目录元数据（如名称）默认走 LWW 自动解。
- 冲突处理后状态：
  - 主版本 `syncStatus=synced`；
  - 备份版本标记 `conflicted_archived`（仅本地可见，不参与常规列表）。

#### G. 边界条件与异常清单

- 登录边界：
  - 未登录允许本地目录与文件 CRUD；
  - 未登录触发同步时仅提示登录，不执行网络写操作。
- 网络边界：
  - 同步中断（断网/超时）时，已成功项保留成功状态，未完成项保持原状态或标 `failed`；
  - 重试不应重复创建远端实体（依赖 `clientId/checksum` 幂等）。
- 数据边界：
  - 目录重名、文件重名、非法文件名需前置校验并阻止提交；
  - 空文件（0B）与超大文件按配置策略处理（允许/禁止需在实现期固定）；
  - 目录删除前若包含文件，需提示“递归删除风险”并二次确认。
- 并发边界：
  - 同一浏览器多标签同时编辑同一目录或文件时，以最后写入本地者为准，并在下次同步进入冲突判定；
  - 同步任务执行中再次点击同步，应做串行化（排队）或按钮禁用。
- 删除边界：
  - 本地删除后未上行前可撤销（可选）；
  - 已同步文件删除时需支持策略：
    - 仅本地删除（云端保留）；
    - 本地+云端同时删除（需明确确认）。

#### H. 测试用例最小集合（建议纳入提测单）

- 用例 1：新建 2 级目录并上传 3 个文件 -> 上行成功 -> 状态全 `synced`。
- 用例 2：下行拉取云端新增目录与文件 -> 本地正确新增并可打开。
- 用例 3：本地与云端同时改同名文件内容 -> 触发冲突并按策略处理。
- 用例 4：上行过程中断网 -> 失败项可重试且不重复创建。
- 用例 5：未登录点击同步 -> 不改本地数据，仅提示登录。
- 用例 6：执行“仅本地删除”与“本地+云端删除”分别验证结果。

#### I. 实现补充约定（2026-04 会话落地）

- 二进制存储链路：
  - 前端：`drive_files` 仅存元数据，文件二进制写入 IndexedDB `blobs`，以 `localBlobRef` 关联；
  - 后端：云盘文件以二进制落盘保存，`drive_files` 仅保存元数据 + `storageId`；
  - 下载：优先本地 `blobs`，本地缺失时回源云端下载流，再转为文件下载。
- 删除同步（已同步文件）：
  - 本地删除后写入删除待同步记录（tombstone）；
  - 下次“同步到云端”先执行远端删除，成功后再清理 tombstone；
  - 远端已不存在（404）按成功处理，避免阻塞后续同步。
- 拉取合并补充：
  - 当 `GET /api/drive/folders` 或 `GET /api/drive/files` 返回后，若某本地记录原为 `synced` 但云端已缺失，则自动降级为 `local_only`（保留本地数据，不自动删除）。
- 页面交互补充：
  - 新增文件触发系统文件选择器（支持多选）；
  - 文件列表提供“下载”按钮；
  - “打开同步面板”以右下角悬浮按钮呈现；
  - 左侧目录标题与目录项显示文件夹图标，目录头部提供拉取/推送 icon。
- 用户提示补充：
  - 若本次拉取触发“云端缺失回退”为本地文件/目录，页面需 toast 提示回退数量（目录数、文件数）。

### 6.1.4 图文编辑

- 支持文字输入。
- 支持图片插入方式：
  - 粘贴（Ctrl+V）
  - 拖拽
  - 文件选择
- 图片在本地可预览、删除、排序（排序可后置）。

### 6.1.5 本地存储（IndexedDB）

- 所有笔记、文件夹、图片元信息默认写入本地库。
- 页面刷新后数据保留。
- 网络断开不影响查看和编辑。

### 6.1.6 搜索与筛选

- 按标题/正文关键词搜索本地笔记。
- 按同步状态筛选（仅本地、待上传、失败等）。
- 按 `sourceDomain` 快速筛选网页短文本条目。
- 支持按条目类型筛选（账号/密码/ID/待办）。
- 笔记搜索与站点信息搜索分开执行，默认互不干扰。

### 6.1.7 上传中心与手动同步

#### 推送（与既有「上传」一致）

- 操作入口：
  - 上传当前笔记
  - 上传全部未同步笔记（`local_only + dirty + failed`）
  - 上传站点信息区未同步短文本（同上状态语义）
- 推送流程：
  1. 先上传图片
  2. 再上传笔记正文与图片 URL 引用
- 推送结果：
  - 成功：状态改 `synced`，回写 `cloudId`
  - 失败：状态改 `failed`，可重试
- 推送中显示进度与结果提示。

#### 拉取（新增）

- 用户主动触发「从云端拉取」：需登录。
- 拉取范围（MVP 建议）：
  - 云端已有笔记与短文本列表及元数据；
  - 笔记关联图片：按需下载到本地 `blobs` 表并更新 `images.uploadedUrl` / 本地引用。
- 与本地合并规则（MVP）：
  - 以「云端 `updatedAt` vs 本地 `updatedAt`」比较；较新一方覆盖较旧一方，或标记冲突待用户选择（实现期二选一，文档先行约定**优先较新时间戳**）。
  - 纯本地未上传条目（`local_only`）与云端无对应 `clientId` 时：保留本地，不删除。
- 登录后的关系规则：
  - 未登录：本地与云端完全隔离，不触发自动同步。
  - 登录成功后：自动执行一次拉取，将云端内容合并到本地。
  - 本地有、云端无：保持为仅本地（`local_only`），不自动删除。
  - 云端有、本地无：直接拉取到本地并标记为 `synced`。
  - 云端与本地都有但内容差异：同时保留两份（本地原文 + 云端副本），由用户后续处理。
- 拉取失败：提示原因，本地数据不丢失，可重试。

#### 云端文件列表入口（新增）

- 在笔记文件名列表区域提供「云端文件列表」入口，支持跳转到云端已同步内容管理页。
- 云端列表中删除文件时，需支持“仅删本地”与“本地+云端同时删除”两种模式。

#### 离线导出 / 导入（新增）

- **导出**：一键导出本地全部笔记及相关图片（以及可选：文件夹、站点信息区短文本、站点空间等，与 PRD 数据模型一致）。
  - 输出为可传输的归档（如单文件 `.mynotes.json` + 侧车资源，或 zip 内含 manifest + `blobs/`），需包含版本号与校验信息，便于后续格式演进。
- **导入**：在目标设备选择导出文件，解析后写入本地 `IndexedDB`（导入策略：默认**合并**；可选「清空本地后全量覆盖」需二次确认）。
- 导入过程展示进度；大文件需防止主线程阻塞（Web Worker 或分块写入，实现期处理）。

### 6.1.7A 与云端能力的关系

- **推送 / 拉取**依赖后端列表、下载、鉴权等接口（见 §6.3）。
- **导出 / 导入**完全本地完成，可用于无网络或不愿使用云端的迁移场景；若目标设备再执行推送，则与现有云端流程衔接。

### 6.1.8 站点信息区与扩展数据一致性

- **主路径（跨端一致）**：扩展与 Web 使用**同一登录体系与后端**；用户在同一账号下通过**手动拉取**合并云端数据、通过**手动推送**上传待同步内容，短文本（`snippets`）在两端遵循**相同的合并与冲突规则**（实现见 `@my-notes/shared` 纯函数与 `@my-notes/sync-client`）。
- **存储说明**：扩展与 Web 各自使用浏览器 **IndexedDB（Dexie，库名与表结构与 Web 一致）**；因**源站不同**（`chrome-extension://` 与 `https?://`），物理上为**两份独立本地库**，「打通」指**云端与同步语义一致**，而非直接读取对方页面的 IndexedDB。
- Web 端站点信息区展示与扩展同一账号下的短文本条目，按 `domain` 分组。
- **离线导出 / 导入**用于无网迁移或备份，不作为扩展与 Web 对齐的主路径。
- Web 端仍保留完整笔记功能；扩展 UI 以短文本为主，底层可与 Web 共用同一套本地库 schema 与全量拉取逻辑，以便与上传中心行为一致。

## 6.2 浏览器扩展功能

- **账号**：支持登录 / 退出（与 Web 相同认证接口）；未登录可本地编辑短文本，**推送与拉取需登录**。

- **同步**：提供**手动推送**（待上传短文本，及与 Web 一致的全量笔记/图片拉取链路若已实现则一并执行）与**手动拉取**（从云端合并到本地）；**不做自动实时同步**（与 PRD 全局策略一致）。
- 功能范围以网页短文本数据 CRUD 为主：
  - 新建：文本录入/粘贴并保存短文本条目
  - 查询：按 `domain` 查看与搜索条目
  - 修改：编辑条目类型与内容
  - 删除：删除指定条目
- 自动携带来源信息：`url / domain / title / captureTime`。
- 对网页短文本提供类型选择：账号/密码/ID/待办/自定义。
- 构建与运行需配置 **API 基址**（如 `VITE_API_BASE`），以便扩展内 `fetch` 使用绝对 URL 访问后端。
- 暂不支持：
  - 展示笔记目录
  - 展示笔记内容
  - 一键保存当前网页全文

## 7. 业务流程

### 7.1 本地编辑流程

1. 用户创建/编辑笔记。
2. 前端实时写入 `IndexedDB`。
3. 状态设为 `local_only` 或 `dirty`。

### 7.2 手动推送（上传）流程

1. 用户点击上传。
2. 若未登录，先登录。
3. 上传图片 -> 上传笔记（及短文本，若在范围内）。
4. 成功回写 `cloudId` 并标 `synced`；失败标 `failed` 并可重试。

### 7.2A 手动拉取流程

1. 用户点击「从云端拉取」。
2. 若未登录，先登录。
3. 请求云端列表与增量/全量数据；图片下载写入本地 `blobs` 与 `images`。
4. 与本地按时间戳（或既定冲突策略）合并；更新界面与同步状态。
5. 失败则提示，本地保留，可重试。

### 7.2B 离线导出 / 导入流程

1. **导出**：用户选择「导出全部」→ 打包本地结构化数据与图片 → 保存为文件。
2. **导入**：用户在另一设备选择文件 → 校验版本 → 合并或覆盖写入 `IndexedDB` → 完成提示。
3. 导入后笔记状态可为 `local_only` 或保留原 `syncStatus`（实现期约定，建议导入后为 `local_only` 或 `dirty` 以便再次推送）。

### 7.3 扩展短文本 CRUD 流程

1. （可选）用户登录；登录成功后可将 token 置于扩展安全存储（如 `chrome.storage.session`），并按需执行**手动拉取**以与云端对齐。
2. 用户在扩展内选择站点后创建短文本条目。
3. 扩展将条目写入 **IndexedDB**（`snippets`），并按 `sourceDomain` 分组展示。
4. 用户可在扩展内直接编辑或删除条目；编辑后同步状态按 Web 相同规则变更（如 `synced` → `dirty`）。
5. 用户**手动推送**后，Web 端同一账号下可通过拉取或上传中心看到一致数据；反之亦然。

### 7.4 网页地址隔离流程（短文本）

1. 用户进入“站点信息区”，在某网页上下文新增短文本条目（如账号、密码、ID、待办）。
2. 系统自动写入 `sourceDomain`，并可附加 `sourceUrl`。
3. 前端在“站点分组视图”中仅展示该域名下数据。
4. 用户切换站点后，默认只看到对应站点的数据集合。

## 8. 数据需求（逻辑模型）

### 8.1 本地（IndexedDB）

- **Web 与扩展**：均使用 Dexie 管理 IndexedDB，**同一库名与版本迁移链**（`my_notes_v1` 及表定义与 `apps/web` 历史实现一致，实现归口 `@my-notes/local-db`）。扩展与 Web 各有一份物理数据库，通过云端同步保持一致。

- `folders`
  - id, name, parentId, updatedAt
- `notes`
  - id, folderId, title, contentText, updatedAt, syncStatus, cloudId, dirty
- `images`
  - id, noteId, localBlobRef, checksum, uploadedUrl
- `clips`
  - id, content, sourceUrl, sourceDomain, sourceTitle, createdAt, status
- `snippets`
  - id, type, content, sourceDomain, sourceUrl, createdAt, updatedAt, syncStatus, cloudId, dirty
- `site_spaces`
  - id, sourceDomain, displayName, createdAt, updatedAt

### 8.2 云端（Server）

- `cloud_notes`
  - id, userId, clientNoteId, title, contentText, updatedAt
- `cloud_note_images`
  - id, noteId, imageUrl, checksum
- `cloud_clips`（如需跨端保留）
- `cloud_snippets`
  - id, userId, clientSnippetId, type, content, sourceDomain, sourceUrl, updatedAt

## 9. 交互与状态规范

- 笔记列表需显示同步状态标识：
  - 仅本地 / 已同步 / 已修改待上传 / 上传失败
- 上传失败提示需包含：
  - 失败原因（网络/鉴权/服务异常）
  - 重试入口
- 对已同步笔记再次编辑后，立即转为 `dirty`。
- 删除操作默认本地软删除（回收站可后置）。
- 涉及网页短文本的列表默认按 `sourceDomain` 进行隔离展示。
- 同一条短文本必须绑定一个 `sourceDomain`，不允许空域名入库。
- 信息架构约束：站点信息区与笔记区为平级一级入口，默认不互相展示数据。

## 10. 非功能需求

- **性能**：常规 1000 条笔记场景下列表与检索可用。  
- **可靠性**：上传失败不丢本地数据。  
- **可用性**：核心操作（新建、编辑、上传）路径清晰。  
- **安全性**：HTTPS 传输、鉴权校验、用户数据隔离。  
- **兼容性**：主流 Chromium 内核浏览器优先。

---

## 11. 指标体系（MVP）

- 上传成功率（成功上传次数 / 上传尝试次数）
- 失败重试成功率
- 扩展端短文本 CRUD 使用率
- 扩展端短文本与 Web 端一致率
- 短文本条目创建数（按类型）
- 站点隔离视图使用率（按域名筛选触发次数）
- 日活跃编辑用户数
- 人均本地笔记创建数

---

## 12. 风险与应对

- **IndexedDB 容量与图片占用风险**：限制单图大小，提供压缩策略（后置）。  
- **多设备本地数据不一致**：通过「手动推送 + 手动拉取」建立跨端通路；离线场景用「导出 / 导入」迁移。  
- **上传冲突风险**：MVP 使用“最后一次手动上传为准”，后续补冲突提示机制。
- **敏感短文本风险（如密码）**：MVP 需明确提示本地存储属性，并预留本地加密能力（后续版本实现）。

---

## 13. 版本里程碑（MVP）

- **M1**：Web 本地笔记闭环（文件夹、图文、IndexedDB）
- **M2**：上传中心闭环（手动推送、状态回写、失败重试）+ 手动拉取闭环 + 离线导出/导入闭环
- **M3**：扩展短文本 CRUD 闭环（扩展端独立管理，Web 端同步可见）
- **M4**：稳定性与指标埋点

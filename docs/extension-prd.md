- 与 Web 端共用能力采用“引用”方式，不在本文重复展开实现细节。

## 2. 能力复用声明（引用 Web，不重复）

- **认证与会话**：复用 `PRD.md` `6.1.1`（登录、退出、未登录行为）。
- **手动推送 / 手动拉取语义**：复用 `PRD.md` `6.1.7`。
- **跨端一致性与冲突处理**：复用 `PRD.md` `6.1.8` 与 `6.1.3B E3`（LWW + 可审计保留）。
- **本地/云端字段与状态**：复用 `PRD.md` `8.1`、`8.2` 与 `syncStatus` 定义（`local_only/synced/dirty/failed`）。
- **实现差异说明**：扩展端仅渲染短文本面板 UI，不展开 Web 页面级模块（如笔记树、图文编辑区）。

---

## 3. 产品定位与范围

- 扩展端定位为“站点短文本速记与同步面板”，以 `snippets` 为核心对象。
- 扩展端独立承载短文本 CRUD 与手动同步入口，不承载 Web 笔记编辑体验。
- 登录后与 Web 共用同一云端账号与数据；本地仍是扩展侧独立 IndexedDB（与 Web 结构一致）。
- 不支持项（与 Web 保持边界）：
  - 展示笔记目录
  - 展示笔记正文编辑器
  - 一键保存整页网页

---

## 4. 扩展面板布局（从上到下）

### 4.1 整体容器（抽屉形态）

- 扩展端 UI 以“侧边抽屉”方式展示，默认从浏览器右侧滑出，覆盖当前页面部分区域。
- 抽屉采用纵向单列布局，自上而下固定为：顶部栏 -> 操作栏 -> 条目列表标题 -> 条目列表。
- 抽屉在窄宽度场景仍保持上述顺序，不允许模块换位。

### 4.2 顶部栏

- 左侧：站点地址展示区（当前页 `domain`，可附带精简 URL）。
- 右侧：账号动作 icon 区：
  - 未登录：显示登录 icon。
  - 已登录：显示退出、拉取、推送 icon（按“退出/拉取/推送”顺序排列）。
- 登录态切换后，icon 区即时刷新，不需要关闭抽屉重开。

### 4.3 操作栏

- 左侧为搜索输入框（可占满操作栏主要宽度）。
- 输入框行为：根据输入内容实时过滤“当前条目列表”（标题、内容、名称均可命中）。
- 过滤仅影响当前站点分组的数据视图，不跨站点检索。

### 4.4 条目列表标题栏

- 左侧固定文案：`条目列表`。
- 右侧新增 icon：点击后创建新条目（进入新增态或打开编辑态）。
- 新增入口在滚动列表时保持可见（标题栏吸顶或固定在列表上方）。

### 4.5 条目列表项

- 每个条目由两行信息区组成：
  - 第一行：条目内容展示区。要求保留并正常渲染多行换行（`\n` 按换行显示，不强制单行省略）。
  - 第二行左侧：条目名称、条目同步状态、更新时间。
  - 第二行右侧：复制、置顶、编辑、删除 icon 操作区。
- 第二行信息建议弱化视觉层级（次级文字样式），第一行内容保持主视觉优先。
- 操作 icon 需支持可点击态与禁用态（例如同步中临时禁用编辑/删除，避免并发冲突）。

---

## 5. 交互规则补充（扩展端专属）

- 置顶：置顶条目在当前站点列表内优先排序显示，且不改变其同步状态语义。
- 复制：复制内容为第一行完整文本（含换行），复制成功需有轻提示。
- 编辑：进入内联编辑或弹层编辑均可，但保存后必须更新 `updatedAt` 并按规则刷新 `syncStatus`。
- 删除：需二次确认（尤其是 `password` 类型），删除后的云端策略沿用 `PRD.md` `6.1.7` / `6.1.8`。
- 拉取/推送执行中，顶部 icon 与列表项操作要有明确 loading 反馈，防止重复触发。

---

## 6. 技术与配置约束

- 扩展端请求后端需使用绝对 API 基址（如 `VITE_API_BASE`）。
- 本地存储使用 IndexedDB（Dexie），与 Web 共用 **`@my-notes/local-db`**：库名 **`my_notes_v2`**（`NOTES_DB_NAME`），**单版本 `version(1)` schema**，与 Web 表结构一致。
- 若历史版本存在 `chrome.storage` 数据，首次启动执行一次性迁移后再进入列表渲染。
